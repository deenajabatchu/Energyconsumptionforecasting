# 🌟 1. Why This Problem? (Idea Behind Project)

Electricity usage is increasing day by day.  
But the problem is:

- Usage is not constant  
- It changes based on time, season, and human activity  

👉 Example:

- Morning → less usage  
- Evening → high usage  

❌ What happens without prediction?

- Produce more electricity → wastage  
- Produce less → power cuts  

👉 So, we thought:

“Can we predict future electricity usage using past data?”

✅ That is the main idea of the project.

---

# 🔍 2. What Already Exists (Existing Systems)

Before our work, people used:

## 1. ARIMA (Statistical model)
- Works only for simple linear data  

## 2. Basic ML models
- Decision Tree  
- SVM  

❌ Problems:

- Cannot handle complex patterns  
- Cannot understand time dependency  
- Less accuracy  

---

# 💡 3. How We Got the Solution Idea

We observed:

- Electricity data is time-series  
- So we need a model that understands time  

👉 That’s why:

- Use Random Forest → for patterns  
- Use LSTM → for time-based learning  

👉 Idea:

“Let’s try both models and compare which works better.”

---

# 📊 4. Dataset (Starting Point)

We used:

- Source: Fingrid (Finland dataset)  
- Rows: 52,965  

## Columns:
- Timestamp  
- Consumption (MWh)  

👉 This is called Univariate Time-Series Data.

---

# 🧹 5. Data Preprocessing (Cleaning Step)

## What we did:

- Converted timestamp → datetime  
- Sorted data (important for time-series)  
- Handled missing values  
- Removed duplicates  

## 👉 Tools:
- Pandas  
- NumPy  

---

# 🧠 6. Feature Engineering (Making Data Smart)

We created new columns:

- Day  
- Month  
- Weekday  
- Lag features  

## 🔹 Lag Feature Meaning

👉 Example:

Today’s value depends on yesterday.

So we create:

- Consumption(t-1)  
- Consumption(t-2)  

👉 This helps model learn patterns.

---

# 🔀 7. Data Splitting

We split data into:

- Training (80%) → to learn  
- Testing (20%) → to check accuracy  

---

# 🤖 8. Model 1: Random Forest

## What it is:
- Group of decision trees  

## How it works:
- Each tree gives prediction  
- Final output = average of all trees  

## Why used:
- Handles nonlinear data  

---

# 🧠 9. Model 2: LSTM

## What it is:
- Deep learning model (RNN)  

## Special ability:
👉 Remembers past data (memory)

## Example:
- Yesterday → Today → Tomorrow  

## Steps:
- Convert data into sequences  
- Normalize data  
- Train LSTM layers  

---

# ⚠️ IMPORTANT CONCEPT

👉 Both models are:

❌ Not combined  
❌ Not sequential  

✅ They are trained separately.

---

# 📏 10. Evaluation Metrics (How we measure performance)

## 1. MSE (Mean Squared Error)

### Formula:
\[
MSE = \frac{1}{n} \sum (y - \hat{y})^2
\]

👉 Meaning:
- Difference between actual (y) and predicted (ŷ)

---

## 2. RMSE

### Formula:
\[
RMSE = \sqrt{MSE}
\]

👉 Gives error in real units.

---

## 3. R² Score

### Formula:
\[
R^2 = 1 - \frac{SS_{res}}{SS_{tot}}
\]

👉 Meaning:
- How well model fits data
