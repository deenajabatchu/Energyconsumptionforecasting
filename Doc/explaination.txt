# Energy Consumption Forecasting and Visualization for Smart Homes using Data Analytics and Machine Learning

## 1. Why This Problem?

Electricity consumption is increasing continuously due to the growing use of electronic devices, smart appliances, and industrial systems. However, electricity usage is not constant and changes depending on time, season, climate, and human activities.

For example:
- Electricity usage is lower during early morning hours
- Electricity usage becomes higher during evening hours

Without proper forecasting:
- Excess energy production may lead to wastage
- Insufficient production may lead to power shortages

To overcome this issue, the project focuses on predicting future electricity consumption using historical energy usage data.

---

# 2. Existing Systems

Earlier systems mainly used statistical and basic machine learning models for electricity forecasting.

## Statistical Models
- ARIMA

## Machine Learning Models
- Decision Tree
- Support Vector Machine (SVM)

### Limitations of Existing Systems
- Unable to handle complex nonlinear patterns
- Cannot effectively capture time dependency
- Lower prediction accuracy for large time-series datasets

---

# 3. Proposed Solution

The proposed system uses both Machine Learning and Deep Learning techniques for forecasting electricity consumption.

The project applies:
- Random Forest for learning nonlinear relationships
- LSTM for learning time-based patterns and sequential dependencies

Both models are trained separately and their performance is compared to identify the better forecasting approach.

---

# 4. Dataset Information

The dataset used in this project is collected from the Fingrid electricity consumption dataset of Finland.

### Dataset Details
- Total Rows: 52,965
- Type: Univariate Time-Series Dataset

### Original Columns
- Timestamp
- Electricity Consumption (MWh)

---

# 5. Data Preprocessing

Data preprocessing is performed to clean and prepare the dataset before model training.

### Preprocessing Steps
- Converted timestamp values into datetime format
- Sorted the dataset chronologically
- Handled missing values
- Removed duplicate records

### Tools Used
- Pandas
- NumPy

# 6. Feature Engineering

Feature engineering is performed to extract useful information from the dataset.

### Features Created
- Day
- Month
- Weekday
- Lag Features

### Lag Features
Lag features are created using previous electricity consumption values.

Examples:
- Consumption(t-1)
- Consumption(t-2)

These features help the model understand how previous consumption affects future electricity usage.

---

# 7. Data Splitting

The dataset is divided into:
- Training Data (80%)
- Testing Data (20%)

### Purpose
- Training data is used to train the model
- Testing data is used to evaluate model performance

---

# 8. Random Forest Model

Random Forest is a Machine Learning algorithm based on multiple decision trees.

### Working Process
1. Multiple decision trees are created
2. Each tree generates predictions
3. Final prediction is obtained by averaging all outputs

### Advantages
- Handles nonlinear data effectively
- Reduces overfitting
- Provides stable predictions

---

# 9. LSTM Model

LSTM (Long Short-Term Memory) is a Deep Learning model based on Recurrent Neural Networks (RNN).

### Special Feature
LSTM remembers previous information using memory cells, which makes it suitable for time-series forecasting.

### Working Process
1. Data is converted into sequences
2. Data is normalized using MinMaxScaler
3. LSTM layers are trained using sequential data
4. Predictions are generated for future values

### Advantages
- Captures temporal dependencies
- Learns long-term patterns
- Provides better forecasting accuracy

---

# 10. Model Evaluation Metrics

The performance of the models is evaluated using different metrics.

## Mean Squared Error (MSE)

### Formula
\[
MSE = \frac{1}{n} \sum (y - \hat{y})^2
\]

### Terms
- \(y\) = Actual value
- \(\hat{y}\) = Predicted value
- \(n\) = Total number of observations

### Purpose
Measures the average squared error between actual and predicted values.

---

## Root Mean Squared Error (RMSE)

### Formula
\[
RMSE = \sqrt{MSE}
\]

### Purpose
Measures prediction error in the original unit of the data.

---

## R² Score

### Formula
\[
R^2 = 1 - \frac{SS_{res}}{SS_{tot}}
\]

### Terms
- \(SS_{res}\) = Residual Sum of Squares
- \(SS_{tot}\) = Total Sum of Squares

### Purpose
Measures how well the model fits the dataset.
